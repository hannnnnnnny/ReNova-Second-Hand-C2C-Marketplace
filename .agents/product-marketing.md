# NovaCart Product Marketing Context

## Product Overview

NovaCart is a portfolio-grade multi-merchant ecommerce website builder. It lets independent merchants choose a storefront template, configure brand settings, add products, publish a generated store, and manage operations from a protected dashboard.

Product category: ecommerce SaaS, storefront builder, merchant operations platform.

Business model shown in this repository: demo-safe SaaS pricing tiers for a portfolio project. No real billing or payment processing is connected.

## Target Audience

Primary audience for the project itself:
- Hiring managers and engineering teams evaluating full-stack product judgment.
- Recruiters reviewing portfolio work for frontend, backend, and SaaS product roles.
- Small-business ecommerce operators as the imagined buyer/user.

Imagined customer profile:
- Independent retailers, boutiques, thrift sellers, homeware shops, sports gear sellers, and small lifestyle brands.
- Merchants who need a credible storefront and basic commerce operations without stitching together many tools.

## Personas

Merchant owner:
- Wants to launch quickly, look polished, and avoid operational chaos.
- Cares about storefront quality, product setup, checkout confidence, and support/refund handling.

Store operator:
- Manages inventory, product status, orders, promotions, support tickets, and refund requests.
- Cares about clear queues, low-stock warnings, safe status transitions, and fast admin workflows.

Technical reviewer:
- Evaluates whether the project shows real architecture, validation, testing, security awareness, and production-style documentation.

## Problems And Pain Points

- Basic ecommerce demos often look like single-store catalog pages rather than real SaaS products.
- Merchants need both customer-facing storefront polish and back-office workflow depth.
- Demo product data frequently feels thin, generic, or unrealistic.
- Portfolio reviewers need evidence of full-stack systems thinking, not only UI screenshots.

## Positioning

NovaCart is a serious ecommerce builder demo that connects polished generated storefronts with merchant operations: catalog, orders, inventory, promotions, analytics, customer care, and refunds.

Short value proposition:
Launch branded merchant storefronts and run the daily commerce work behind them from one demo-safe platform.

## Messaging Pillars

- Multi-merchant by design: platform pages, merchant onboarding, admin workspace, and generated storefront routes.
- Production-style operations: inventory warnings, order state transitions, promotion targeting, customer records, support tickets, and refund queues.
- Shopper confidence: product variants, wishlist behavior, cart quantity editing, delivery options, checkout validation, order confirmation, support, and refund flows.
- Portfolio credibility: Spring Boot REST API, Vue 3 frontend, MySQL runtime persistence, H2 tests, JWT admin security, docs, screenshots, and repeatable verification.

## Proof Points

- Demo storefronts for fashion, sports, home goods, and minimal boutique merchants.
- Rich demo catalog with categories, collections, variants, stock states, discounts, ratings, review counts, and merchandising labels.
- Protected admin workspace with dashboard, products, categories, collections, promotions, orders, inventory, customers, analytics, support, refunds, settings, template management, and theme editing.
- Tests cover backend API/service behavior and frontend stores/utilities.

## Objections To Address

- "Is this real ecommerce?" Clarify that checkout is intentionally demo-safe and does not process real payments.
- "Is it only a UI demo?" Point to backend entities, DTO validation, transactional order logic, stock movement history, and tests.
- "Can many merchants use it?" Show generated store routes, store switcher, onboarding, template system, and local merchant store creation.
- "Is the data copied from real brands?" Demo names, data, copy, and assets are original and safe.
